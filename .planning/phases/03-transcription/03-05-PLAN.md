<task type="checkpoint:human-verify" gate="blocking">
  <name>Checkpoint 2: Transkription funktioniert und Pipeline-Output korrekt</name>

  <what-built>
    AudioController.onRecordingComplete-Callback → AppDelegate → TranscriptionService.transcribeWithResampling()
    → print("Transkription: \(text)") → appState.resetToIdle()
  </what-built>

  <how-to-verify>
    **Voraussetzung:** Checkpoint 1 abgeschlossen (Modell heruntergeladen, isModelReady = true)

    **Schritt 1:** Xcode-Konsole oeffnen (View > Debug Area > Activate Console)

    **Schritt 2:** Hotkey (⌥⌘R) druecken
    Erwartetes Verhalten:
    - Menu-Bar-Icon wechselt auf rotes "Recording"-Icon (pulsierend)
    - "Tink"-Sound spielt

    **Schritt 3:** 10-15 Sekunden klar auf Deutsch sprechen, z.B.:
    "Dieser Test prueft ob die lokale Transkription korrekt funktioniert."

    **Schritt 4:** Hotkey (⌥⌘R) erneut druecken (oder Stille abwarten)
    Erwartetes Verhalten:
    - "Pop"-Sound spielt
    - Icon wechselt auf blaues "Transcribing"-Icon (kurz sichtbar)
    - Icon kehrt zu Idle-Icon zurueck
    - Xcode-Konsole zeigt: `Transkription: Dieser Test prueft ob die lokale Transkription korrekt funktioniert.`
      (Wortlaut kann leicht abweichen — Inhalt muss erkennbar korrekt sein)

    **Schritt 5: 30-Sekunden-Test (RECORD-04 Success Criteria 3)**
    Hotkey druecken, 30 Sekunden sprechen, stoppen.
    Erwartetes Verhalten:
    - Kein Absturz, kein Hang
    - Transkription erscheint in Konsole nach 5-20 Sekunden (Modell-Inferenzzeit)
    - State wechselt zu .idle
  </how-to-verify>

  <resume-signal>
    Schreibe "ok" wenn alle 3 Schritte erfolgreich waren (Recording-Icon, blaues Transcribing-Icon,
    Konsolen-Output mit deutschen Text). Oder beschreibe den Fehler genau.
