## Spacing Scale

Deklarierte Werte (Vielfache von 4 pt) — fortgeführt aus Phase 1 `DesignTokens.swift`:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4 pt | Icon-interne Abstände; Waveform-Linien-Gap zum Mic-Symbol |
| sm | 8 pt | Menüpunkt-Innenabstand vertikal; Abstand Label–Control in Settings |
| md | 16 pt | Standard-Element-Abstand; Abstand zwischen Settings-Sektionen |
| lg | 24 pt | Abschnittstrennungen innerhalb SettingsView |
| xl | 32 pt | Fensterkanten-Padding (SettingsView) |

Ausnahmen:
- Menu-Bar-Icon-Canvas: 18 × 18 pt — fest durch NSStatusItem-Constraint (keine Ausnahme von der Skala, da kein Spacing-Token)
- Waveform-Bereich innerhalb Icon: 18 pt breit × 4 pt hoch (unterste 4 pt des 18×18-Canvas = xs)
- Touch-Target für Settings-Controls: mindestens 20 pt Höhe (macOS HIG minimum für Checkboxen und Picker)

## Icon-State Machine Contract

Fortgeführt aus Phase 1, ergänzt um Waveform-Layer:

| Zustand | Icon | Farbe | Pulse | Waveform |
|---------|------|-------|-------|----------|
| `.idle` | `mic.fill` | #8E8E93 | nein | unsichtbar |
| `.recording` | `mic.fill` + Waveform-Linie | `systemRed` | ja (0.8 s) | sichtbar, animiert |
| `.transcribing` | `mic.fill` | `systemBlue` | nein | unsichtbar |
| `.llmProcessing` | `mic.fill` | `systemPurple` | ja (1.2 s) | unsichtbar |

**Waveform-Linie (D-01 bis D-04 aus CONTEXT.md):**
- Position: unterhalb des `mic.fill`-Symbols, nicht überlagernd
- Canvas-Bereich: 18 pt breit × 4 pt hoch (unterste 4 pt des 18×18-Icon-Canvas)
- Abstand Mic-Symbol zur Waveform: 0 pt (direkt angrenzend, kein Gap)
- Form: horizontale Kurve aus N Liniensegmenten, amplitudenmoduliert durch RMS-Pegel (0.0–1.0)
- Minimalamplitude (Stille): 1 pt — Linie bleibt sichtbar, keine Null-Linie
- Maximalamplitude: 4 pt (füllt den gesamten Waveform-Bereich)
- Linienbreite: 1 pt (`lineWidth: 1`)
- Farbe: `Color(.systemRed)` (identisch mit Recording-Zustandsfarbe)
- Update-Rate: aus AVAudioEngine-Tap-Callback, über Observation Strategy B (`updateIcon()`)
- Parameter-Name in StatusBarIconView: `audioLevel: CGFloat` (Bereich 0.0–1.0, RMS normiert)
- Anzahl dargestellter Samples: dem Entwickler überlassen (innerhalb 18×18-Constraint)

**Pulse-Animation:** unverändert aus Phase 1 (easeInOut, opacity 1.0 ↔ 0.5, repeatForever).

## Interaction Contract

### Hotkey-Aufnahme-Toggle (RECORD-01)

| Aktion | Auslöser | Reaktion |
|--------|----------|----------|
| Aufnahme starten | Hotkey drücken (⌥⌘R Standard) | Icon → `.recording`; Waveform sichtbar; Start-Ton |
| Aufnahme manuell stoppen | Hotkey erneut drücken | Icon → `.transcribing`; Waveform unsichtbar; Stopp-Ton |
| Aufnahme auto-stoppen | Stille ≥ konfigurierter Schwellwert | identisch mit manuellem Stopp (D-07: selber Ton) |

### Audio-Cues (FEED-02, D-05 bis D-07)

| Ereignis | NSSound | Charakter |
|----------|---------|-----------|
| Aufnahme startet | Diskreter System-Ton (z.B. „Tink") | Hell, kurz (~150 ms) |
| Aufnahme stoppt (manuell oder Auto) | Diskreter System-Ton (z.B. „Pop") | Tiefer als Start-Ton, kurz (~150 ms) |

Genauer NSSound-Name: Claude's Discretion (aus CONTEXT.md). Beide Töne müssen sich
klar voneinander unterscheiden. Kein AVAudioEngine-Ton; kein Bundle-Asset.

### Mikrofon-Auswahl (RECORD-03, SET-04)

| Element | Spezifikation |
|---------|---------------|
| Platzierung | SettingsView — eigener Abschnitt „Mikrofon" |
| Control | `Picker` (SwiftUI) mit `pickerStyle(.menu)` |
| Label | „Eingabegerät" |
| Optionen | Enumerierte `AVCaptureDevice`-Liste; Default = System-Standard |
| Hilftext | 11 pt Caption unter dem Picker: „Wähle das Mikrofon für die Aufnahme." |

### Stille-Erkennungs-Einstellung (SET-03)

| Element | Spezifikation |
|---------|---------------|
| Platzierung | SettingsView — eigener Abschnitt „Stille-Erkennung" |
| Control | `Slider` (SwiftUI), Bereich 0.5 s – 5.0 s, Schrittweite 0.5 s |
| Standardwert | 1.5 s (D-09) |
| Label | „Automatischer Stopp nach" |
| Wertanzeige | Rechts neben Slider: „{wert} s" (formatiert als String, z.B. „1,5 s") |
| Hilftext | 11 pt Caption: „Aufnahme stoppt automatisch, wenn für diese Dauer keine Sprache erkannt wird." |

### Mikrofon-Berechtigung (D-13, D-14)

| Zustand | UI |
|---------|-----|
| Berechtigung erteilt | Kein Banner — normaler Settings-Inhalt |
| Berechtigung verweigert | Roter Banner am oberen Rand von SettingsView |
| Banner-Text | „Mikrofonzugriff verweigert" (Label-Größe, weiß auf systemRed) |
| Banner-Subtext | „Öffne die Datenschutz-Einstellungen, um VoiceScribe Zugriff zu erteilen." (Caption, weiß) |
| Banner-Button | „Einstellungen öffnen" — öffnet `NSWorkspace.shared.open(privacyURL)` |
| Banner-Padding | sm (8 pt) innen, md (16 pt) seitlich |
| Banner-Höhe | intrinsisch (min. 44 pt) |
