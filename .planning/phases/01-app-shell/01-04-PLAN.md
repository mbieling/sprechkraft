    App ist gestartet (siehe Task 1).

    Test A: Linksklick-Cycle
    1. Klicke EINMAL mit linker Maustaste auf das Menu-Bar-Icon
       ERWARTET: Icon wird ROT (systemRed). Es pulsiert sanft (Opacity 1.0 ↔ 0.5, 0.8s-Rhythmus).
    2. Klicke EINMAL mit linker Maustaste
       ERWARTET: Icon wird BLAU (systemBlue). Es pulsiert NICHT — statisch.
    3. Klicke EINMAL mit linker Maustaste
       ERWARTET: Icon wird LILA (systemPurple). Es pulsiert sanft, langsamer als Recording (1.2s-Rhythmus).
    4. Klicke EINMAL mit linker Maustaste
       ERWARTET: Icon wird wieder GRAU (#8E8E93). Statisch.

    Test B: Hotkey-Cycle (SET-02)
    5. Wechsle in eine beliebige andere App (z.B. Safari, Notes, Terminal)
    6. Drücke ⌥⌘R (Option+Command+R) — OHNE SPRECHKRAFT-Fenster/Menü zu öffnen
       ERWARTET: Icon in der Menüleiste wechselt auf rot (Recording). Pulsiert.
    7. Drücke erneut ⌥⌘R
       ERWARTET: Icon wechselt auf blau.
    8. Wiederhole zwei weitere Drücker — Icon muss durch lila und zurück zu grau wandern.

    Test C: Animation-Qualität
    9. Lass den Recording-Zustand für 5 Sekunden stehen und beobachte die Pulse-Animation. Sie muss GLEICHMÄSSIG laufen, ohne Ruckeln, ohne sichtbare Aussetzer.
    10. Lass den LLM-Zustand für 5 Sekunden stehen. Die Pulse muss SICHTBAR LANGSAMER sein als Recording.

    FEHLSCHLAG falls: Farben weichen ab, Animation ruckelt, Hotkey reagiert nur wenn SPRECHKRAFT fokussiert ist, Zustandsreihenfolge ist nicht idle→recording→transcribing→llmProcessing→idle.
  </how-to-verify>
  <acceptance_criteria>

    - User bestätigt: Pulse-Animation nur für rot (Recording) und lila (LLM)

    - User bestätigt: Animation läuft flüssig (mind. 5 Sekunden ohne Ruckeln)

  <verify>Manuelle Bestätigung durch User ("approved"). Unterstützend: `sfltool dumpbtm` oder `osascript -e 'tell application "System Events" to get the name of every login item'` listet SPRECHKRAFT bei aktiviertem Toggle.</verify>

  </read_first>
  <what-built>
    LaunchAtLogin-modern-Library (SMAppService unter der Haube) persistiert den Toggle-Zustand systemweit. AppDelegate zeigt im Menü den aktuellen Zustand als NSMenuItem.state (.on / .off) und toggelt via LaunchAtLogin.isEnabled.toggle().
  </what-built>
  <how-to-verify>
    Test A: Toggle visuell
    1. Öffne Rechtsklick-Menü. Merke dir den aktuellen Zustand von "Beim Login starten" (Haken oder kein Haken).
    2. Klicke "Beim Login starten" — Menü schließt sich.
    3. Öffne erneut das Rechtsklick-Menü
       ERWARTET: Der Haken hat sich umgekehrt.

    Test B: Systembestätigung
    4. Öffne Systemeinstellungen → Allgemein → Anmeldeobjekte (oder auf Deutsch: Login Items)
       ERWARTET: SPRECHKRAFT taucht dort als Anmeldeobjekt auf (oder verschwindet, je nach Toggle-Richtung).

    Test C: Persistenz über App-Neustart
    5. Aktiviere den Toggle (Haken erscheint).
    6. Wähle "Beenden" im Menü.
    7. Starte die App erneut (z.B. Xcode Run oder durch Klick auf die .app).
    8. Öffne das Rechtsklick-Menü
       ERWARTET: "Beim Login starten" zeigt NOCH den Haken. Der Zustand überlebte den Neustart.

    Test D: Persistenz über macOS-Reboot (optional, falls Zeit)
    9. Aktiviere Toggle, beende App.
    10. Starte macOS neu.
    11. NACH dem Login: SPRECHKRAFT muss automatisch laufen (Icon in Menüleiste sichtbar).

    FEHLSCHLAG falls: Haken ändert sich nicht, Anmeldeobjekte-Liste zeigt keinen Eintrag, Zustand geht beim App-Neustart verloren, App startet nach Reboot nicht automatisch trotz aktivem Toggle.
  </how-to-verify>

    - User bestätigt: Test B — SPRECHKRAFT erscheint in Systemeinstellungen → Anmeldeobjekte
    - User bestätigt: Test C — Zustand überlebt App-Neustart
    - Test D ist OPTIONAL — User vermerkt "getestet" oder "übersprungen"
